# webpack 打包原理

>webpack 中每个模块有一个唯一的 id，是从 0 开始递增的。  
整个打包后的 bundle.js 是一个匿名函数自执行。  
参数则为一个数组。数组的每一项都为个 function。  
function 的内容则为每个模块的内容，并按照 require 的顺序排列。

>识别入口文件,识别模块依赖，来打包代码。webpack做的就是分析代码,转换代码，编译代码，输出代码


## 如何实现一个简单的webpack
- 读取文件分析模块依赖
- 对模块进行解析执行(深度遍历)
- 针对不同的模块使用相应的loader
- 编译模块，生成抽象语法树AST。
- 循环遍历AST树，拼接输出js。
